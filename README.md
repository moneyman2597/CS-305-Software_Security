# CS 305 – Software Security Portfolio

## Overview

This repository contains my **Artemis Financial Secure Software Report**, where I analyzed vulnerabilities and implemented secure coding practices to improve application security.

---

## Client & Requirements

Artemis Financial is a financial services company that required improved software security to protect sensitive customer data. The primary goal was to identify vulnerabilities in their application and implement solutions to ensure secure communication and data protection.

---

## Security Analysis & Strengths

One of my strengths in this project was identifying outdated dependencies and insecure coding practices. I applied secure coding principles such as input validation and encryption to reduce risk.

Secure coding is critical because it prevents data breaches, protects user information, and maintains trust. Strong software security also helps companies avoid financial loss and reputational damage.

---

## Challenges & Learning

A challenging part of this project was understanding how different vulnerabilities impact system security and selecting the most effective mitigation strategies. This experience helped me improve my ability to think critically about security risks.

---

## Security Enhancements

To increase layers of security, I:

* Implemented secure communication using HTTPS
* Updated vulnerable dependencies
* Applied input validation to prevent injection attacks

In the future, I would use tools like:

* OWASP Dependency-Check
* Static code analysis tools
* Dynamic testing tools

---

## Testing & Validation

After refactoring the code, I tested the application to ensure it remained functional. I also re-ran vulnerability scans to confirm that no new vulnerabilities were introduced.

---

## Tools & Practices

* OWASP Dependency-Check
* Java secure coding practices
* Encryption (SHA-256, HTTPS)
* Static analysis techniques

---

## Portfolio Value

This project demonstrates my ability to:

* Identify and analyze software vulnerabilities
* Apply secure coding practices
* Use industry tools to improve application security

This is a strong example of my skills in software security and DevSecOps practices.
